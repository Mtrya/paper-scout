# OpenSkill Deep-Dive Notes

## What the paper is actually doing

OpenSkill addresses open-world self-evolution: an LLM agent must improve after deployment starting from only a task prompt, with no curated skills, successful trajectories, verifier signals, or ground-truth answers. The agent must build both its skills and its own verification signals from scratch, using only open-world resources (documentation, repos, web pages). Target-task supervision is reserved strictly for final evaluation.

The key insight is decoupling: skills are explicit, transferable artifacts (not model weights), and verification comes from self-built virtual tests grounded in independently checkable facts retrieved from the open world.

## What is genuinely novel

1. **Open-world knowledge acquisition as the primary skill source.** Prior methods (Skill Creator, AutoSkill, Memento) rely on parametric knowledge or interaction traces. OpenSkill uses a deep-research agent (Gemini) to retrieve current, domain-specific knowledge from the open web and synthesize a structured skill plan.

2. **Leakage-free virtual verifier.** A separate LLM session generates pytest suites anchored to verification knowledge (reference values, statistical invariants, cross-validation procedures, documented output formats). This verifier never sees ground-truth tests. The paper shows 88.9% coverage of GT test intents and statistically significant alignment with GT outcomes (OR=2.97, p=0.035).

3. **Diagnostic-driven targeted retrieval.** When virtual tests fail, a classifier determines whether the failure is a bug (self-fixable) or a knowledge gap (needs additional retrieval). This prevents unnecessary API calls and focuses retrieval on genuine gaps.

4. **Cross-model transfer without adaptation.** Skills generated by Opus 4.6 transfer to Haiku, Qwen, DeepSeek, and Mistral with consistent gains (5.5–14.8 points over no-skill baseline).

## Core method at near-reimplementation detail

**Stage 1 — Knowledge acquisition:**
- Query synthesis: an LLM agent planner reads the task instruction, metadata, and environment preview, emits a research query requesting APIs, signatures, examples, and pitfalls.
- Leakage filtering: benchmark names and variants are stripped from queries.
- Retrieval: sanitized query goes to Gemini Deep Research (asynchronous, up to 1h timeout). Result is a background document k_i with deduplicated source URLs.
- Skill planning: a Skill Planner decomposes k_i into 1–4 skills (name, responsibility, key functions, dependencies).
- Verification knowledge: a second retrieval pass (D^v) asks for four classes of checkable anchors: reference values, statistical invariants, cross-validation procedures, and known input-output pairs.

**Stage 2 — Leakage-free skill evolution:**
- Initial skill generation: the base agent drafts skill set S_i^(0) based on (task, env, plan, knowledge).
- Virtual test generation: an Independent Verifier (fresh LLM session, no shared context with creator) generates a pytest suite using k_i^v as the "primary oracle."
- Iterative refinement: up to J=3 rounds. At each round, skills are executed and scored against virtual tests. If pass rate < 1.0, a structured diagnostic is produced and the skill is revised. If the diagnostic indicates a knowledge gap, targeted retrieval is triggered (max 3 searches).
- Termination: exits when virtual pass rate = 1.0 or J rounds exhausted.

**Stage 3 — Zero-shot target evaluation:**
- The final skill set is deployed to the target agent (which may differ from the construction agent).
- Ground-truth tests are evaluated; no GT signal was used during construction.

## Experimental evidence and credibility

**Setup:** Three benchmarks (SkillsBench with 11 domains, SocialMaze, ScienceWorld), two target agents (Opus 4.6 / Claude Code, GPT 5.2 / Codex). Seven closed-world baselines: No Skill, Self-Gen, CoT, Skill Creator, AutoSkill, Memento, SkillNet.

**Results:**
- SkillsBench: OpenSkill achieves 43.6% (Opus) and 42.1% (GPT), beating the best baseline by +8.9 and +8.8 points. Human upper bound is 44.5% / 44.8% — OpenSkill is within 1–3 points.
- SocialMaze: 82.7% (Opus), 70.7% (GPT) — best automated method.
- ScienceWorld: 90.0% (Opus), 85.3% (GPT) — best automated method.
- Transfer: OpenSkill skills consistently improve all four weaker target models.

**Red flags:**
- Cost is substantial: ~$1,800 per full benchmark run (84 tasks) at Opus 4.6 pricing. Per-task: ~749K tokens and ~39 min for skill creation, ~400K tokens and ~91 min for GT evaluation.
- Virtual verifier precision is only 56.9%. False positives (25/84 tasks) cluster in "high-accuracy near-misses" and "partial correctness" — the verifier catches structural validity but misses deep semantic errors.
- Manufacturing domain collapses to 0% for all automated methods, suggesting the benchmark has tasks that are genuinely beyond current LLM agent capabilities.
- Heavy API dependency: Gemini Deep Research, Gemini Flash, Claude Opus, GPT-5.2. This is not a lightweight or offline-capable system.
- Code repository is currently a placeholder.

## Related-work situating

**Skill Creator (Anthropic, 2025):** Iteratively refines skills from parametric knowledge and self-graded tests. OpenSkill adds open-world retrieval and an independent verifier, yielding +8.9 points over Skill Creator.

**AutoSkill (Yang et al., 2026):** Experience-driven skill bank from interaction traces. Collapses on GPT 5.2 (24.7%→11.2%). Its skills are model-coupled and fail to transfer.

**Memento (Zhou et al., 2026):** Memory-based RL framework with Read-Write-Reflective learning. Better than AutoSkill but still closed-world. OpenSkill beats it by +3.5 (SkillsBench Opus) and +3.3 (GPT).

**Voyager (Wang et al., 2023):** Open-ended embodied agent with skill library. Operates in Minecraft with environment feedback. OpenSkill generalizes to any task domain and does not require environment rewards.

## Research action loop

**Question:** Does the virtual verifier actually drive skill quality, or is most of the gain from open-world retrieval alone?

**Action:** Figure 4b ablates the two components. Open-world query alone: +6.1 points. Virtual verifier alone: +6.3 points. Both together: +8.2 points over parametric-only baseline. The components are largely complementary.

**Interpretation:** Both matter. Open-world query provides correct knowledge; the verifier catches implementation errors. Neither alone suffices.

**New question:** Why does performance peak at 3 refinement iterations and degrade at 5–10? Figure 4a shows 82.7% at 3 iterations, 79.9% at 5, 78.0% at 10. The paper attributes this to overfitting to virtual feedback. This suggests the virtual verifier becomes a misleading optimization target with too many iterations — a known risk in self-supervised refinement.

**Blocker for reproduction:** The pipeline relies on commercial deep-research APIs (Gemini Deep Research) that are not easily replaceable with local models. Reproducing the exact results would require access to the same API versions.

## Bottom-line judgment

**Read.** OpenSkill makes a clean conceptual contribution: it formalizes open-world self-evolution and shows that open-world retrieval plus self-built verification can bootstrap a learning loop without target-task supervision. The cross-model transfer is a genuinely useful property. However, the cost is high, the virtual verifier is imperfect, and the system is tightly coupled to commercial APIs. The framework is more important than the specific numbers — the numbers will change as base models improve, but the three-stage pipeline (acquire → verify → deploy) is a durable design pattern for agent self-improvement.
