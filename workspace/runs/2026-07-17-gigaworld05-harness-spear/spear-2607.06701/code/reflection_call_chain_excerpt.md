## 4. The actual invocation — cpp/unreal_plugins/SpCore/Source/SpCore/UnrealUtils.cpp:164-258

```cpp
std::map<std::string, SpPropertyValue> UnrealUtils::callFunction(const UWorld* world, UObject* uobject,
                                                                 UFunction* ufunction, const std::map<std::string, std::string>& args, ...)
{
    uint32 num_bytes = ufunction->ParmsSize;                                  // reflection: param struct size
    std::vector<uint8_t, SpAlignedAllocator<uint8_t, 4096>> args_vector(num_bytes, 0);
    ufunction->InitializeStruct(args_vector.data());                          // default-construct params
    for (TFieldIterator<FProperty> itr(ufunction); itr; ++itr) {              // walk UFUNCTION params via reflection
        FProperty* property = *itr;
        if (property->HasAnyPropertyFlags(EPropertyFlags::CPF_Parm)) { ... build SpPropertyDesc (name -> value_ptr) ... }
    }
    for (auto& [property_name, property_desc] : property_descs) {
        if (Std::containsKey(args, property_name)) {
            setPropertyValueFromString(property_desc, args.at(property_name)); // UE JSON import: Python dicts -> C++ values
        }
    }
    {
        TGuardValue<bool> guard_value(GAllowActorScriptExecutionInEditor, true);
        uobject->ProcessEvent(ufunction, args_vector.data());                  // THE call — same dispatch path as Blueprints
    }
    ... re-read all params out of the buffer (catches out-params + return value), serialize back to strings ...
}
```

KEY POINT: there is NO code generator and no per-function bindings anywhere in the repo.
One generic path (find UFunction by name string -> fill its reflected param struct -> ProcessEvent)
exposes every UFUNCTION visible to UE's runtime reflection system. The "14,485 functions" figure
is a *runtime count* over the loaded reflection registry, produced by examples/get_class_info/run.py,
not a static binding set.
